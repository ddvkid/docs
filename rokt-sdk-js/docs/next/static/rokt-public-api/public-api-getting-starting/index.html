Getting started with Rokt Public API

Where to start

To start, you need to register a new Client App with the Rokt platform. To do this, you will need to contact your Rokt Account Manager to request that a new Client App with a unique name be registered for you.
As a result you will receive two pieces of information. A "Client ID" and "Client Secret". These are confidential pieces of information about your app and you need to keep them in a safe place.

Here is a sample Client Id and Client Secret:

ClientId: "40svbin0d194subpohl079rhck"
Secret: "1dimhvr1v6skae9uhvtgs3chs2astnjf0469df6ul9hurubtoovn"

You need to use these client credentials in the REST interactions with the Rokt public API.

Get Access Token

To call any endpoint on the Rokt Public API, a security token is needed. Let's call it Access Token. Access tokens allow your app to call endpoints on the Rokt Public API. In addition, access tokens allow Rokt to identify client apps, the type of data that each client app is accessing , and prevent malicious apps from accessing data that belong to other clients.

Access tokens are generated based on the 'Client ID' and 'Client Secret' created in the previous step. The access token lasts one hour. During that hour the security token can be used to call all endpoints on the Rokt Public API. Before it expires, you will need to re-generate the access token based on the client credentials.

In order to get the access token, an endpoint is exposed in the Rokt Public API:

POST https://api.rokt.com/auth/oauth2/token

Request Parameters in Header

HeaderDescription
Authorization“Client Id” and “Client Secret” must be passed in the authorization header through Basic HTTP authorization. The content of the header is Basic Base64Encode(client_id:client_secret).
Content-TypeMust always be application/x-www-form-urlencoded.

Request Parameters in Body

ParameterDescription
grant_typeMust be client_credentials.

Examples Request with a Positive Response

Sample request

POST https://api.rokt.com/auth/oauth2/token&
Content-Type='application/x-www-form-urlencoded'&
Authorization=Basic xxx

grant_type=authorization_code
