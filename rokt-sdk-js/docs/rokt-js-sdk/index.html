Rokt JavaScript API

Rokt object allows integration with various Rokt solutions and communication between client and Rokt solution. For the main entry point and more details, please see Rokt interface.

Changelog

All notable changes to this project will be documented in this file.

0.3.34 - 0.3.49

No public API changes

0.3.33 - 2019-10-18

  • Added Rokt.setFulfillmentAttributes that allows setting attributes which can be sent to the server only when the user accepts an offer.

0.3.27 - 0.3.32

No public API changes

0.3.23 - 2019-09-04

  • Added additional FailureReasons to Rokt.onFailure covering scenarios when we did not receive placements or offers so partner can decide to handle the scenario gracefully

0.3.11 - 0.3.22

No public API changes

0.3.10 - 2019-05-01

  • Added Rokt.closeAll to remove all placements on the page.

0.3.6 - 0.3.9

No public API changes

0.3.5 - 2019-02-28

Changed

  • Cart.getItems returns a copy of a Rokt cart state.
  • Rokt.Attributes allows to pass locale value that will be used to determine language to be used in displayed placements.
  • Rokt.Attributes allows to pass country code that can be used to determine language to be used in displayed placements.
  • Rokt.RoktConfiguration allows pageIdentifier to be defined on Rokt.init to uniquely identify the page where your placement appears.

0.3.3 - 0.3.4

No public API changes

0.3.2 - 2018-12-17

Added

  • Added Rokt.setAttributes superseding Rokt.setConsumerData. New method's name better reflects its purpose and aligns with ROKT Cart API naming.
  • Added Rokt.setAttributes example in Rokt interface documentation

Deprecated

  • Deprecated Rokt.setConsumerData in favour of Rokt.setAttributes

0.3.1 - 2018-12-06

No public API changes

0.3.0 - 2018-11-13

Added

  • Added new possible FailureReasons

Changed

  • Breaking renamed LOADED_CALLBACK FailureReasons to ON_LOADED_CALLBACK to better reflect its purpose.

0.2.0 - 2018-10-18

Added

  • Added catalogItemId to CartItem
  • Added new possible FailureReasons

Removed

  • Rokt.init no longer expects roktTagId as it is retrieved form Rokt Tag Script

Changed

  • Breaking renamed CartItem id to cartItemId. Since cartItemId value is GUID its value can change per a fresh impression.
  • Updated Cart.onUpdateItem method so it reflects possible null values in callback parameters. Nulls may happen in case of user adding or removing item when old or updated item are not available.

0.1.0 - 2018-09-18

Added

  • Initial version of the API.
